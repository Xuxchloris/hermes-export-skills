## 场景二：单客户全链路分析（最常用）

```
【提示词模板 — 替换 { } 中的内容】

我是{折叠露营桌}的外贸业务员，目标市场是{美国}。
请对以下公司做完整的客户分析：

公司网站：{https://example.com}
公司名称：{Example Outdoor Inc.}

请按顺序完成：
1. 背调 — 分析公司业务类型、产品线、客户群、服务地区，输出背调报告
2. 评分 — 按产品匹配度/渠道价值/决策层线索/市场价值/网站质量/风险逐项打分，给出A/B/C/D优先级
3. 如果评分B级以上，写一封120-160词的英文开发信草稿，要求：
   - 包含一个基于网站证据的个性化切入点
   - 包含一个产品价值点
   - 轻量CTA，不用best price/guaranteed/buy now
   - 标注个性化证据来源
   - 添加合规退订行

不编造任何客户信息。所有邮件为草稿，人工审核后发送。
```

## 场景三：批量处理客户名单

```
【提示词模板】

我有一个客户名单文件 {exports/pipeline-demo/pipeline-input.csv}，目标市场是{美国和欧盟}。

请帮我批量处理：
1. 读入CSV，按网站域名去重，规范化列名
2. 标记缺少公司名/网站的记录
3. 标记非目标市场/无关行业的记录
4. 逐行背调 + 评分，按优先级排列
5. 为A级和B级客户生成开发信草稿
6. 输出 prospects.enriched.xlsx、scores.xlsx、email_drafts.xlsx

运行命令参考：
python tools/batch_prospect_pipeline.py --input {your_file.csv} --product {PRODUCT.yaml} --market {MARKET.yaml} --tone {TONE.yaml} --output-dir exports/pipeline

所有输出草稿状态，人工审核后发送。
```

## 场景四：客户回复分析与下一步

```
【提示词模板】

我收到了一封客户回复，内容如下：

---
{粘贴客户回复原文}
---

请帮我分析：
1. 这封回复属于什么类型？（询盘/报价请求/目录请求/样品请求/会议请求/异议/不感兴趣/退订/自动回复/不确定）
2. 买家表达了哪些明确信号？
3. 是否提出了具体需求？产品、数量、目的地是什么？
4. 还有哪些商业条件需要确认？
5. 基于以上分析，建议我下一步做什么？

如果涉及报价/回复邮件，帮我生成草稿。所有输出需人工审核。
```

## 场景五：报价单生成

```
【提示词模板】

请为以下买家生成报价草稿：

买家：{Demo Outdoor Inc.}
国家：{United States}
SKU：{CT-200A}
数量：{500}
贸易术语：{FOB Ningbo}

要求：
1. 从配置中读取产品规格和对应数量的阶梯价格
2. 生成报价表，含：编号、日期、SKU、品名、规格、包装、数量、单价、金额、术语、付款方式、交期、有效期
3. 数量低于MOQ时标记并说明
4. 数量高于MOQ时按阶梯价计算
5. 导出HTML和Excel格式
6. 如需PDF从HTML生成

报价为草稿，人工审核确认后方可发送。
```

## 场景八：跟进计划

```
【提示词模板】

我{5月23日}给一家{A级}客户发了开发信，目前没有回复。
客户：{Example Distributors Inc.}
优先级：{A}
最后联系：{2026-05-23}
状态：已开发信，未回复

请帮我制定跟进计划：
1. 在首次发信后第3、7、14天设置跟进节点
2. 每个节点注明任务类型和原因
3. 需要新邮件草稿的节点标注
4. 如果需要草稿，一并生成

所有任务为提醒，人工审核后执行。
```

## 提示词写作原则

基于 hermes-export-skills 的设计哲学，好的外贸 Agent 提示词应满足：

1. **明确输入源**：产品、价格、市场、语气从哪里读取
2. **指定输出格式**：JSON、Excel、HTML，还是纯文本
3. **确定边界规则**：什么可以做，什么不能编造
4. **标注状态**：draft / blocked / needs_review
5. **串联流程**：背调 → 评分 → 开发信 → 跟进

反例（不好的提示词）：
```
帮我开发一个美国客户
```
→ 太模糊，Agent 不知道从哪一步开始

正例（好的提示词）：
```
产品是折叠露营桌CT-200A，目标美国市场。
先帮我背调 https://example-outdoor.com 这家公司，
如果适合就写英文开发信草稿。
不编造任何客户信息。
```
→ 明确任务、顺序、边界
