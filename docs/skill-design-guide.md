# Skill Design Guide

Good trade skills should be:

- Config-driven: no product, price, or market hard-coding.
- Evidence-backed: company claims need a source.
- Human-reviewed: email and quotation outputs are drafts.
- Narrow: each skill handles one job.
- Composable: outputs should be structured enough for the next skill.

Required sections in every `SKILL.md`:

- `Overview`
- `When to Use`
- `Inputs`
- `Outputs`
- `Procedure`
- `Verification`
- `Common Mistakes`
